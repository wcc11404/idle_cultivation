# 客户端定位与边界

## 当前模式

- 客户端为展示与交互编排层。
- 服务端为业务真值与最终结算来源。
- 客户端不再作为业务真值来源，不维护独立权威结算闭环。

## 客户端职责

- 发起业务请求并处理结构化响应（`reason_code + reason_data`）。
- 维护必要运行态与 UI 体验（如按钮状态、战斗面板状态、日志输出）。
- 在少量链路保留体验型本地逻辑（乐观更新、预扣、回放），并最终与服务端状态对齐。

## 服务端职责

- 状态校验、业务判定、结算与结果返回。
- 返回稳定语义码和必要结构化数据。
- 不负责客户端中文文案细化与 UI 呈现风格。

## 客户端-服务端契约

- 业务提示文案由客户端模块内映射，不依赖服务端 message 文本。
- 技术错误仍由网络层统一处理（网络异常、鉴权失效、被顶号、5xx 等）。
- 全量状态同步通过 `GET /game/data` 与对应列表接口完成。
