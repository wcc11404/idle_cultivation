# 日志系统接入规范

## 目标

- 客户端日志统一三分类：`【系统】`、`【战斗】`、`【生产】`。
- 新模块接入日志时，先决定分类，再决定文案，不允许绕过 `LogManager` 另起一套输出口径。

## 分类与入口（唯一真值）

- 系统日志：`log_manager.add_system_log(message)`
- 战斗日志：`log_manager.add_battle_log(message)`
- 生产日志：`log_manager.add_production_log(message)`

禁止：
- 新增 `add_xxx_log` 第四种分类。
- 继续使用历史“炼丹独立分类”或透传旧标签。

## 模块归属规则

- `【战斗】`
  - 历练战斗时间轴、战斗结算、战斗掉落、塔层挑战结果。
- `【生产】`
  - 炼丹：开始、每轮 report 成功/失败、停止、收丹汇总。
  - 采集：开始、每轮 report 成功/失败、停止。
- `【系统】`
  - 其余全部（登录、设置、储纳、术法、修炼、任务、网络技术错误等）。

## 文案规则

- 战斗掉落：
  - 用战斗专属文案，例如：`战斗胜利，获得A、B`
  - 不使用储纳“获得物品xx”口径。
- 生产掉落：
  - 采集成功：`采集成功，获得A、B`
  - 采集失败：`采集失败，本轮未获得产物`
  - 炼制成功：`炼制成功，获得A`
  - 炼制失败：`炼制失败，返还材料A`（有返还时）
  - 汇总：`收丹停火：成丹X枚，废丹Y枚`

## 去重与静默入包

- 若某流程已经输出战斗/生产专属掉落日志，入背包时必须静默，避免重复出现“获得物品xx”。
- 当前静默入口：
  - `GameUI.begin_silent_item_added_logs()`
  - `GameUI.end_silent_item_added_logs()`

## 日志筛选 UI 契约

- 日志区筛选标签固定四个：`全部 / 系统 / 战斗 / 生产`。
- 切换标签只过滤显示，不清空日志池。
- 标签选择只在当前登录会话内保留；重新登录默认回到“全部”。

## 容量与裁剪

- 日志池采用“全局单池上限”策略，默认上限 `500`。
- 超限时裁剪最旧日志，不按分类单独保留。

## 新系统接入清单（提交前自查）

1. 模块日志是否全部走三分类之一。
2. 是否存在与储纳“获得物品”重复的掉落日志。
3. 是否补了对应 GUT 用例（至少验证一条成功文案和一条失败文案）。
4. 是否同步更新模块文档与本规范（若改了口径/分类）。
