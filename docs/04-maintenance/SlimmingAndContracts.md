# 瘦身与契约维护规范

## 瘦身原则

- 不改变现有体验链路：修炼乐观更新、炼丹预扣、历练回放。
- 删除“确定无调用”的旧逻辑优先；存疑项先隔离再删。
- 每批清理后执行 `./run_tests.sh` 回归。

## 外部注入契约字段

部分字段可能在模块内不直接读取，但由 `GameUI` 或初始化流程注入，视为契约字段。

- 处理原则：不能仅因“本文件内无引用”而删除。
- 只有在同步改造调用方并完成回归后，才能下线。

## 文案与错误处理规范

- 业务提示统一由模块根据 `reason_code + reason_data` 翻译。
- 技术错误（网络/鉴权/服务器异常）由网络层统一处理。
- 禁止恢复到“直接透传服务端业务 message”模式。

## 文档与代码一致性

- 文档描述“当前行为”，不保留历史行为并列表述。
- 每次改动提示策略、状态流、测试入口时，必须同步文档。
