# 设置与通用提示

## 职责与边界

- 管理设置页中的改昵称、排行榜、登出。
- 业务失败文案由模块映射，技术失败由网络层兜底处理。

## 关键状态

- 设置主面板与排行榜子面板切换状态。
- 顶部账号展示数据来源：`GameManager.account_info`。

## API 交互

- `POST /auth/change_nickname`
- `GET /game/rank`
- `POST /auth/logout`

## 功能触发流转

### 1) 打开设置页

1. 显示设置主面板，默认隐藏排行榜面板。
2. 绑定按钮事件（登出、确认改昵称、查看排行榜、返回）。

### 2) 点击确认改昵称

1. 先做本地基础校验（空值、长度、空格、全数字）。
2. 调 `auth/change_nickname`。
3. 成功：
   - 输出“昵称修改成功”。
   - 更新 `GameManager.account_info.nickname`。
   - 调 `game_ui.update_account_ui` 同步顶部显示。
4. 失败：按 reason_code 映射原因（敏感词、非法字符等）。

### 3) 点击排行榜

1. 切换到排行榜子面板。
2. 调 `game/rank`。
3. 成功：
   - 清空旧列表并重建头部 + 条目。
   - 有数据时默认静默，不额外打印“加载成功”日志。
   - 空数据时输出“排行榜暂无数据”。
4. 失败：输出“排行榜加载失败”映射文案。

### 4) 点击返回

1. 从排行榜子面板回到设置主面板。
2. 不触发额外网络请求。

### 5) 点击登出

1. 调 `auth/logout`（尽力调用）。
2. 无论接口是否失败，都执行本地 token 清理。
3. 切回登录场景，保证用户可重新登录。

## reason_code 文案策略

- 改昵称：reason_code 精确映射，避免底层错误直接展示。
- 登出：失败只做必要提示，不阻止本地退出。
- 排行榜：有数据成功静默；空数据或失败时提示。

## 失败处理与回退

- 网络/鉴权异常由 NetworkManager 统一处理。
- 排行榜失败不影响昵称与登出操作。

## 测试覆盖点

- 改昵称成功与非法昵称映射。
- 排行榜成功静默、失败提示。
- 登出本地 token 清理与场景切换。

## 典型触发链路（函数级）

以“点击排行榜”为例：

1. `SettingsModule.on_rank_pressed` 切换到排行榜子面板并调 `api.get_rank`。
2. 成功时 `SettingsModule._render_rank_list` 重建列表；有数据成功保持静默。
3. 空列表时输出“排行榜暂无数据”。
4. 失败时按 `reason_code` 输出失败提示，不展示技术细节。
5. 点击返回由 `on_rank_back_pressed` 仅切面板，不触发额外请求。
