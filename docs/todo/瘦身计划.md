**目标与不变约束**
- 保持现有玩家体验不变：修炼 pending/flush 的乐观更新、炼丹预扣池与返还、历练依赖服务端回放的流程全部保留。
- 不改服务端接口与 reason_code 语义，不改当前 UI 文案映射。
- 未使用的逻辑直接删除；疑似在用但无引用先隔离为 `_legacy_*`，观察无回归后再删。

**基线与守护**
- 基线测试：`./run_tests.sh`（GUT 全量）。
- 视改动范围按需跑 API 冒烟：修炼 start/report/stop/突破、炼丹 start/report/stop、背包 use/expand/discard/organize、术法 equip/upgrade/charge、历练 simulate/finish、改昵称/排行榜。

**发现未用逻辑的办法**
- `rg --files-with-matches`/`rg -l` 搜索关键词：`_local_`、`_offline`、`_cache`、`save_game`、`load_game`、`message`、`reason`、`optimistic`、`old_`、`backup`。
- `rg -n "func" scripts` 后用 `rg -n <method_name>` 查调用；无引用且测试通过视为可删。
- 记录初步清单到本文件，分“可删 / 可隔离 / 保留”。

**分模块清理指引**
- 网络/存储层：删除本地存盘、旧 token 兜底、`GameServerAPI` 里 message/reason fallback、未用离线重置逻辑。
- GameUI/GameManager：移除未用 save/load、离线累积、背景刷新定时；保留 `refresh_all_player_data` 主链及乐观/预扣触点。
- CultivationModule：只保留乐观计数与 flush；删本地 preview/反作弊模拟/旧日志/旧 message fallback。
- AlchemyModule：保留预扣与返还；删本地结算/模拟/旧日志/多余锁；不动进度条与 API 上报点。
- ChunaModule（背包）：删重复本地可用性校验、空礼包本地判断、旧文案 fallback；保留 reason_code 映射与 UI 刷新。
- SpellModule：删本地资源/次数校验与 message fallback；保留 reason_code 映射、战斗中锁定提示。
- LianliModule：保留 API 回放；删本地战斗模拟、旧反作弊/重置残留、未用日志。
- Login/Settings：删旧本地账号缓存/存盘路径；保留自动登录、改昵称、排行榜现逻辑。
- core 数据/工具：移除旧本地权威期模拟函数和未引用工具，保留静态配置表与资源翻译。

**隔离策略**
- 处理存疑代码：重命名为 `_legacy_*`，从初始化链路断开；在此文件标注来源与回退方式；一轮测试稳定后再删。

**工作节奏**
- 每一批清理后立即跑 `./run_tests.sh`；触及网络层再补一次 API 冒烟。
- 发现测试回归立刻回滚本批改动或把问题代码移回主链并标注原因。

**验收标准**
- GUT 全绿，无新增警告/泄漏日志。
- 乐观修炼、炼丹预扣、历练回放行为与现版本一致。
- 无旧 message/reason fallback 仍在产出 UI 文案。
- `_legacy_*` 只作为临时隔离且无初始化入口。

**注意事项（本轮新增）**
- 模块公开成员字段（如 `player/spell_system/alchemy_system`）虽可能在模块内部未直接读取，但仍可能被 `GameUI` 直接赋值，属于“外部注入契约”。
- 这类字段不能按“文件内无引用”直接删除，后续统一先标注为“契约字段”，除非同步改造 `GameUI` 注入方式并完成回归。
