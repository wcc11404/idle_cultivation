## 3. 战斗结算口径

## 3.1 伤害计算公式

基础伤害改为：

```text
伤害 = max(攻击 * (1 - max(防御-穿透,0) / max(防御-穿透+K, K)), 1) * 系数
```

解释：
- `攻击`：攻击方最终攻击
- `防御`：防守方最终防御
- `穿透`：攻击方最终穿透
- `K`：防御折算常数，当前固定取 `100`
- `系数`：主动术法或普通攻击对应的伤害倍率
- 最终伤害最低保底为 `1`

## 3.2 命中判定

```text
是否命中 = 判定（命中 - 闪避）
```

当前实现口径：
- 我方与敌方初始命中率都为 `100%`
- 命中判定使用：`我方命中率 - 敌方闪避率`
- 用该结果与随机数比较决定是否命中
- 若未命中，则本次伤害直接为 `0`

## 3.3 暴击判定

```text
是否暴击 = 判定（暴击 - 抗暴）
```

当前实现口径：
- 我方与敌方初始暴击率、抗暴率都为 `0%`
- 暴击判定使用：`我方暴击率 - 敌方抗暴率`
- 用该结果与随机数比较决定是否暴击

## 3.4 暴击伤害

```text
触发暴击后的伤害 = 伤害 * (1 + 爆伤)
```

解释：
- `爆伤` 是额外伤害倍率，不是最终倍数本身
- 例如：`爆伤 = 100%` 时，暴击后伤害为 `2.0 倍`

## 4. 术法系统总览

## 4.1 术法分类

术法分为 4 大类：

- 吐纳
- 攻击
- 开局
- 生产

说明：
- 吐纳类：主要作用于修炼或资源积累相关机制
- 攻击类：主要作用于战斗中的主动伤害与附加效果
- 开局类：战斗开始时直接生效的属性或状态加成
- 生产类：当前只是预留类型，后续可用于炼丹、采集、制造等生产玩法增强

## 4.2 五行属性

每个术法都带五行属性之一：

- 金
- 木
- 水
- 火
- 土
- 无

说明：
- “无” 表示无属性术法
- 当前文档只确定术法自带五行标签，五行相克或五行联动效果暂未展开

## 4.3 稀有度

每个术法都带稀有度：

- 凡
- 黄
- 玄
- 地
- 天

当前约束：
- 稀有度只区分掉落概率与获取难度
- 稀有度本身不直接决定最终强度上限
- 真正强度由模板参数、等级、星级、属性加成、技能效果共同决定

## 4.4 等级与星级

每个术法包含：
- 等级：`1-10`
- 星级：`0-5`

其中：
- 等级提升：复用现有升级条件机制
  - 使用次数
  - 灵气消耗
  - 其他现有升级条件
- 星级提升：新增独立升星机制

## 4.5 升级与升星的区别

### 升级

升级会提升：
- 属性加成
- 技能效果

### 升星

升星只提升：
- 属性加成

升星不会提升：
- 技能效果

### 升星消耗

升星需要：
- 对应术法解锁道具
- 升星材料 `空白玉简`

当前规则：
- 升几星就需要几个对应术法解锁道具
  - 升 `1星` 需要 `1` 个
  - 升 `2星` 需要 `2` 个
  - 升 `3星` 需要 `3` 个
  - 依此类推
- `空白玉简` 从 `2星 -> 3星` 开始，每次升星固定需要 `1` 个

## 10. UI 升级需求

## 10.1 术法缩略卡

当前正式实现口径：

- 正式术法页一行显示 `4` 个术法缩略卡，超出后自动换行，只纵向滚动，不横向滚动，不显示滚动轴。
- 详情入口统一为点击整张术法缩略卡；旧的 `查看` 按钮已废弃。
- 点击打开详情与滑动判定分离，滑动超过阈值后不触发详情。
- 五行底托、图标、文字都不拦截输入，点击这些区域也会触发整卡详情。
- 非生产术法底部只显示一个居中加宽的 `装备 / 卸下` 按钮；未获取时显示禁用 `装备`。
- 生产术法不显示底部按钮，但仍可点击整卡查看详情。
- 装备/卸下成功后，仅目标卡片播放一次轻闪反馈；缩略卡内装备按钮不做按压缩放。
- 已装备态只改变边框颜色，不改变边框宽度，避免卡片刷新时视觉跳动。

术法缩略卡从上到下显示：

1. 星级
2. 名字
3. 五行属性图标
4. 稀有度
5. 等级
6. 按钮

布局要求：
- 一行显示 `4` 个术法缩略卡
- 超出后自动换行
- 整个术法缩略卡区域支持纵向滚动
- 不允许横向滚动
- 不显示滚动轴

显示规则：
- `0星`：不显示星级
- `1星`：显示 `1` 个星星
- `N星`：显示 `N` 个星星
- 五行属性不用直接显示文字或 ASCII，使用类似 `🔥` 这类风格的美术图标资源
- 稀有度通过术法名字颜色体现
- 术法名字颜色规则直接复用物品稀有度颜色规则

说明：
- `查看` 与 `装备` 先按两个独立按钮设计
- 五行属性使用项目内正式图标资源，不直接显示文字或 ASCII

```json
    "2": {
      "requirements": {"unlock_item_count": 3, "star_material_count": 1},
      "attribute_bonus": {"attack": 0.002}
    },
    "3": {
      "requirements": {"unlock_item_count": 4, "star_material_count": 1},
      "attribute_bonus": {"attack": 0.002}
    },
    "4": {
      "requirements": {"unlock_item_count": 5, "star_material_count": 1},
      "attribute_bonus": {"attack": 0.002}
    },
    "5": {
      "attribute_bonus": {"attack": 0.002}
    }
  },
  "levels": {
    "1": {
      "use_count_required": 0,
      "spirit_cost": 0,
      "attribute_bonus": {},

```

字段说明：
- `type`：`breathing / active / opening / production`
- `rarity`：`fan / huang / xuan / di / tian`
- `element`：`metal / wood / water / fire / earth / none`

  - `star_material_count` 即使为 `0` 也显式写出，保持和升级条件配置风格一致
- `levels`：保留现有升级条件与每级效果

## 11.2 不同类型术法的 effect 建议

### 吐纳类
吐纳类本轮不新增可运行特殊机制配置，只保留结构承载能力。

### 主动伤害类
`effect` 统一升级为 `List` 结构，每个术法可以同时拥有多个效果。

示例：
```json
[
  {
    "effect_type": "instant_damage",
    "damage_percent_min": 1.2,
    "damage_percent_max": 4.8,
    "trigger_chance": 0.1
  },
  {
    "effect_type": "drain_health",
    "percent": 0.05
  },
  {
    "effect_type": "turn_gauge_delta",
    "value": -0.15
  }
]
```

说明：
- `effect_type` 统一替代旧的 `type`
- `instant_damage` 负责伤害倍率区间
- 恢复造成伤害的气血、敌方行动条变化等，都通过额外 effect 项表达

### 开局类
开局类没有新增新的 effect 结构要求，直接参考现有配置表达方式扩展即可。

### 生产类
生产类没有新增新的 effect 结构要求，直接参考现有配置表达方式扩展即可。

## 11.3 行动条减少效果的战斗挂点

当前服务端历练战斗流程里，玩家释放主动术法的主路径在：
- `app/game/domain/lianli/LianliSystem.py`
  - `_player_action(...)`

在这个函数里，主动术法会调用：
- `spell_system.use_spell(spell_id, player_attributes, enemy_attributes)`

因此“敌方行动条减少”这类效果，当前最合适的落点是：
1. 在 `SpellSystem.use_spell(...)` 中计算术法效果结果
2. 把例如 `turn_gauge_delta = -0.15` 这类结果放进返回值
3. 在 `LianliSystem._player_action(...)` 里接住这个返回值
4. 直接修改敌方当前行动条（例如 `enemy_atb`），并把效果写进 `battle_timeline`

这套挂法的好处是：
- 术法效果定义仍归 `SpellSystem`
- 战斗流程控制仍归 `LianliSystem`
- 不需要把行动条逻辑硬塞进配置解释层
- `effect` 是 `List` 之后，只需在 `use_spell(...)` 中按顺序解释各 effect 项即可
