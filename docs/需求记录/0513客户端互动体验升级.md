# 客户端互动体验升级

## 1. 目标与范围

本轮规划用于系统性提升客户端的“手感”和“氛围感”。前期已经补充了一批更精美的静态素材，例如术法元素图标、货币图标、修炼小人、境界顶栏背景等，但当前客户端整体仍偏静态，常见操作缺少即时反馈，部分关键玩法缺少仪式感。

本需求先做体验设计与分级，不直接要求一次性实现全部内容。

目标：

- 高频操作有轻量、稳定、即时的反馈。
- 关键玩法有符合修仙题材的动效表现。
- 素材、动效、音效都服务于游戏风格，不为“热闹”而堆效果。
- 优先选择 Godot 客户端可低风险实现的方案。
- 宁愿少加，也不要加入廉价、刺耳、简陋或割裂的效果。

一期建议聚焦：

- 内视修炼页
- 通用按钮
- 通用弹窗
- Tab 切换
- 数字变化
- 少量奖励反馈

本期不优先做复杂音效、全屏大招级演出、战斗镜头演出、抽奖完整动画。

---

## 2. 体验原则

### 2.1 动效是反馈，不是流程锁

动效不应阻塞核心游戏流程。尤其是修炼、草药采集、炼丹、历练这类循环玩法，服务端返回成功后应优先推进状态，再异步播放表现反馈。

错误示例：

- 等动画播完才开始下一轮采集。
- 等奖励飞行动画结束才刷新本地数量。
- 高频按钮每次点击都触发长动画。

推荐做法：

- 状态先更新，动效随后播放。
- 动效只影响视觉表现，不控制业务完成时机。
- 高频动效保持在 `0.08s - 0.25s`。

### 2.2 统一封装，不散落到模块里

按钮、弹窗、数字跳动这类通用反馈不应分别写在各业务模块中。建议后续新增统一表现层，例如：

- `UIFeedbackManager`：通用 UI 动效。
- `GameEffectManager`：玩法事件动效。
- `AudioManager`：音效播放、静音、音量、冷却。

业务模块只发出“播放某类反馈”的请求，不直接关心 Tween、粒子、Shader 的具体实现。

### 2.3 移动端优先，克制使用性能开销

Godot 可以高效实现大量 UI 动效，但需要控制使用方式：

- 优先使用 `Tween` 实现轻量缩放、透明度、位移。
- 固定流程演出可使用 `AnimationPlayer`。
- 粒子只用于少量核心场景，不在列表卡片中大量使用。
- Shader 只用于扫光、边框光等少数高价值场景。
- 页面隐藏时，循环动效必须停止或暂停。

## 3. 风格方向

整体视觉风格应延续当前客户端的修仙题材：

- 米色、浅金、玉色、青蓝、暖金为主。
- 光效偏灵气、云雾、符纹、玉石微光。
- 动效要柔和、短促、有呼吸感。
- 稀有反馈可以更强，但不要变成现代霓虹或科幻 UI。

通用动效关键词：

- 轻弹
- 淡入
- 呼吸
- 灵气粒子
- 柔光扩散
- 云雾流动
- 玉石微闪

不推荐：

- 过强抖动。
- 高频全屏闪白。
- 按钮点击时播放长动画。
- 低质量音效或电子蜂鸣感。
- 所有元素同时动。

## 4. 分级升级清单

### 4.1 P0：通用按钮反馈

当前问题：

- 按钮点击反馈主要依赖 Godot 默认状态，手感偏硬。
- 移动端触摸后缺少“已经点到”的即时确认。

建议升级：

- 按下时缩放到 `0.96 - 0.98`。
- 松开后回弹到 `1.0`。
- 禁用按钮不播放点击反馈。
- 重要按钮可附带轻微亮度变化。
- 底部一级 Tab 不做点击缩放，避免导航栏整体产生跳动感；Tab 只做选中态颜色、描边或背景过渡。

实现难度：低。

实现建议：

- 用统一 helper 包装 `Button` 的 `button_down/button_up/pressed`。
- 基于 `Tween`，避免每个按钮单独写重复逻辑。
- 对列表中大量按钮要复用参数，不做复杂动画。

优先覆盖：

- 地区卡片主按钮。
- 历练卡片主按钮。
- 修炼开始/停止按钮。
- 弹窗确认/取消按钮。

### 4.2 P0：弹窗打开与关闭

当前问题：

- 弹窗出现和消失偏硬切。
- 详情弹窗、确认弹窗、奖励弹窗可以有统一模板感。

建议升级：

- 遮罩 `alpha 0 -> 目标透明度`。
- 弹窗主体 `scale 0.96 -> 1.0`。
- 弹窗主体同时 `alpha 0 -> 1`。
- 关闭时反向淡出，时间更短。

实现难度：低。

实现建议：

- 扩展现有公共弹窗模板。
- 所有弹窗默认使用同一套动效。
- 外部点击关闭只触发关闭，不向下穿透。

优先覆盖：

- 术法详情弹窗。
- 物品详情弹窗。
- 确认弹窗。
- 个人信息/改名弹窗。

已落地口径：

- `PopupStyleTemplate` 已正式承接装饰弹窗壳层，后续正式弹窗统一通过模板构建。
- 弹窗统一最小尺寸为 `500 x 350`，不做 `compact / normal / tall` 三档固定尺寸；内容较多时由正文区域滚动或自适应扩展。
- 正式视觉包含米金底板、浅金标题分割线、淡金云纹覆盖层与四角金色角饰；角饰骑在边框外侧，按钮区底部保留安全留白。
- 云纹透明度当前为 `0.40`，并使用保持比例覆盖策略，避免高弹窗中纹样被纵向拉伸。
- 已迁移：术法详情、个人信息、邮件详情/确认、储纳丢弃确认、储纳批量使用，以及两个调试预览场景。
- 新增弹窗不要再手写小尺寸裸 `Panel`；如果业务内容确实很少，也应接受统一最小尺寸，保证角饰、纹样和点击规则一致。

### 4.3 P0：Tab 切换反馈

当前问题：

- Tab 页面切换偏突然。
- 切换后玩家不容易感知页面已经刷新完成。

建议升级：

- 新页面 `alpha 0 -> 1`。
- 内容轻微上移或下移 `6px - 10px` 后归位。
- Tab 按钮选中状态加轻微高光过渡。

实现难度：低到中。

实现建议：

- 由统一导航层触发，不在各 Tab 模块内分别实现。
- 页面首次进入和重复切换可使用同一套动效。
- 数据刷新不应等待切换动效完成。

优先覆盖：

- 内视。
- 储纳。
- 术法。
- 地区。
- 历练。
- 设置。

### 4.4 P0：内视修炼页强化

当前优势：

- 已有新的修炼小人素材，包含未修炼与修炼中两张状态图。
- 小人已经可点击触发修炼按钮逻辑。

当前问题：

- 修炼中状态仍可进一步强化生命感。
- 修炼状态切换的反馈不够突出。
- 灵气数值每秒变化一次，不适合每次都播放闪烁或放大，否则会显得频繁和吵。

建议升级：

- 修炼中小人做轻微呼吸缩放，范围 `1.00 -> 1.025`。
- 修炼中小人周围的粒子效果作为谨慎可选项，不作为一期必做；如果实现，必须使用与当前小人素材一致的柔和灵气风格，并先做静态/动态预览确认不违和。
- 开始修炼时，小人外圈出现一次柔光扩散。
- 停止修炼时，粒子淡出，小人平稳回到未修炼状态。
- 灵气数值常规秒跳不播放动效；只在开始修炼、停止修炼、突破、离线奖励结算等低频关键变化时做轻微反馈。

实现难度：中。

实现建议：

- 呼吸缩放使用循环 Tween。
- 如后续启用粒子，粒子数量控制在几十以内，并使用同色系低透明度贴图。
- 页面隐藏或切换 Tab 时暂停循环动效。
- 不改修炼业务流程，只监听状态变化。

优先级理由：

- 内视是核心首页。
- 修炼是最符合“修仙感”的长期动作。
- 这里的体验提升最容易让玩家感知游戏变精致。

### 4.5 P1：红点状态保持静态

当前结论：

- 红点只作为状态提示，不在一期加入出现、消失、弹跳等动效。
- 红点通常位于导航、入口卡片、任务/邮件提示附近，过度动效会分散玩家注意力。
- 后续除非出现明确的状态感知问题，否则红点保持当前稳定显示方式。

实现难度：无。

实现建议：

- 不新增红点动效。
- 只维护红点位置、层级和显示/隐藏准确性。

### 4.6 P1：货币与物品数量变化

当前问题：

- 灵石、仙晶、物品数量变化时比较静态。
- 玩家完成操作后缺少“资源变化已发生”的即时感。

建议升级：

- 数字变化时标签轻微放大。
- 增加时使用暖金色闪一下。
- 减少时使用轻微下沉或短暂暗色。
- 不做复杂飞行动画作为一期默认方案。

实现难度：低到中。

实现建议：

- 由数据绑定层或 UI 状态刷新层比较新旧值。
- 只对当前可见的货币/物品数量播放。
- 储纳格子数量变化时只动对应格子，不重建全屏动效。

落地记录：

- 右上角灵石、仙晶在 `GameUI.update_ui()` 中比较新旧值，首次初始化不播放；数量变化时对应数字轻弹，并在右侧显示格式化后的 `+/-` 变化量。
- 储纳页在 `ChunaModule.update_inventory_ui()` 中维护当前可见格子的数量快照；同一格、同一物品数量变化时，只对该格数量角标播放轻弹反馈。
- 当物品数量变化后数量角标为空（例如从 `x2` 变为单个物品）时，改为轻闪对应格子，避免全屏或整页动效。
- 当前选中的物品详情数量变化时，只对详情里的数量文本播放轻弹反馈。
- 首次构建储纳格子、首次进入页面、隐藏页面时不播放数量变化反馈，避免不可见或初始化阶段产生无意义动效。

### 4.7 P1：奖励获得反馈

当前问题：

- 获得物品主要依赖日志，缺少轻量视觉确认。

建议升级：

- 普通奖励：日志 + 小型浮层提示。
- 高稀有奖励：物品名按稀有度颜色闪光。
- `地/天` 稀有度：可增加一次小范围光环扩散。

实现难度：中。

实现建议：

- 先不做复杂“物品飞入背包”。
- 先做轻量奖励 toast 或当前区域内的短提示。
- 与稀有度颜色系统保持一致。

适用场景：

- 离线奖励。
- 历练结束。
- 草药采集每轮成功。
- 炼丹完成。
- 未来天机阁抽奖结果。

### 4.8 P2：地区卡片与历练卡片动效

当前优势：

- 地区与历练页已改为统一卡片模板，视觉基础较好。
- 已落地：卡片主按钮继续复用通用按钮反馈，锁定态按钮不播放强动效。
- 暂不落地：卡片结构目前在主场景初始化时提前创建，首次错峰淡入大多发生在不可见页面，实际收益不明显，避免保留无感动效。

建议升级：

- 卡片进入页面时轻微错峰淡入。
- 主按钮点击时使用通用按钮反馈。
- 卡片锁定态不做强动效，只保留稳定灰态。

实现难度：中。

注意：

- 卡片列表较多时，不要每次刷新都全量播放入场动画。
- 仅首次进入页面或列表结构变化时播放。

### 4.9 P2：术法页轻量反馈

当前优势：

- 术法卡片已有元素图标、稀有度颜色和详情弹窗。
- 已落地：术法缩略卡正式页改为四列布局，降低 5 列卡片在手机纵向界面里的拥挤感。
- 已落地：缩略卡使用稀有度色条/边框、已装备徽章、右上角 `★N` 星级徽章。
- 已落地：移除缩略卡 `查看` 按钮，详情入口统一为点击整张卡片；五行底托、图标和文字不拦截整卡点击。
- 已落地：非生产术法只保留居中加宽的 `装备/卸下` 按钮；生产术法不显示底部按钮。
- 已落地：装备/卸下成功后，仅目标术法缩略卡播放一次轻闪反馈；缩略卡内装备按钮不做按压缩放，避免造成卡片放大的错觉。
- 已落地：升级成功后，若详情弹窗打开，则弹窗主体播放一次轻闪反馈；升星成功当前不播放弹窗闪烁，避免面板隐藏/残影风险。
- 已落地：上述反馈复用 `UIFeedbackManager`，不为动效额外触发术法列表全量重建。

后续约束：

- 未获得或不可操作状态不播放强反馈。
- 术法卡片点击本身不额外闪烁，避免滚动列表中显得吵。
- 卡片内部新增装饰节点时，默认不要拦截输入，除非它本身就是按钮。
- 已装备态只改变边框颜色，不改变边框宽度，避免卡片池刷新时布局跳动。

实现难度：中。

注意：

- 术法列表重建成本曾经影响体感，应避免为了动效增加全量重建。
- 如果详情弹窗打开，只刷新必要显示与动效。

## 5. 音效策略

### 5.1 一期不强行加入复杂音效

当前没有稳定的高质量音效来源。为了避免加入廉价、刺耳、割裂的声音，一期不建议强行上线完整音效系统。

音效原则：

- 没有合适素材就不加。
- 不使用来源不明的网络音效。
- 不加入明显电子蜂鸣、廉价合成声。
- 音效必须支持设置页静音开关。

### 5.2 可接受的轻量音效范围

如果后续有合适素材，可优先加入：

- 按钮轻点：玉石/木质短点击。
- 成功确认：短铃音。
- 错误提示：低频短钝音。
- 获得奖励：柔和灵光音。

暂不建议一期做：

- 完整背景音乐。
- 战斗全过程音效。
- 大量角色技能音效。
- 高频循环音效。

### 5.3 音效接入要求

后续如接入音效，应统一由 `AudioManager` 管理：

- 静音状态。
- 音量配置。
- 音效冷却，避免连续点击爆音。
- 音效资源预加载。
- 测试或调试环境可关闭。

## 7. 推荐实施顺序

### 阶段一：通用低风险手感

目标：用最少成本让所有页面“点起来更舒服”。

包含：

- 通用按钮点击反馈。
- 通用弹窗开关反馈。
- Tab 切换淡入。

验收：

- 高频点击不卡顿。
- 动效不会阻塞请求和 UI 刷新。
- 旧测试全绿。

### 阶段二：内视修炼强化

目标：把首页核心循环做出修仙氛围。

包含：

- 修炼小人呼吸动效。
- 开始/停止修炼状态切换动效。
- 低频关键数值变化反馈。

验收：

- 修炼状态切换明显。
- 页面隐藏后循环动效停止。
- 长时间运行无明显性能问题。
- 如果启用粒子，必须先确认与修炼小人素材画风一致；不一致则宁愿不加。

### 阶段三：奖励与稀有反馈

目标：让获得资源、稀有物品、关键成功有仪式感。

包含：

- 轻量奖励提示。
- 稀有度颜色光效。
- 历练、采集、炼丹成功反馈。
- 为未来天机阁抽奖预留高稀有表现规范。

验收：

- 普通奖励不吵。
- 高稀有奖励有明显差异。
- 不引入复杂音效依赖。

- 弹窗打开关闭无穿透、无残影。
- Tab 切换不影响数据刷新。
- 修炼页长时间停留无明显掉帧。
- 页面切走后循环粒子和 Tween 不继续空转。
- 设置静音后，未来音效完全不播放。

性能观察：

- 动效不应造成明显请求返回后的 UI 延迟。
- 高频列表刷新不应全量重播入场动画。
- 粒子数量可控，不在不可见页面运行。

## 9. 暂不做事项

一期暂不做：

- 全量音效系统上线。
- 背景音乐。
- 战斗技能复杂演出。
- 抽奖完整动画。
- 红点出现/消失/弹跳动效。
- 修炼灵气数值每秒跳动反馈。
- 所有页面卡片入场动画。
- 大量 Shader 高光。
- 复杂物品飞行动画。

这些内容不是不能做，而是需要等基础交互层稳定后，再按玩法价值逐步补充。
